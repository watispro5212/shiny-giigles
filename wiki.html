SHARD

An isolated bot process instance. Sharding allows the Nexus to scale across thousands of sectors without performance degradation.

LATENCY

The temporal delay in data transmission pulses. Measured in milliseconds (ms), lower latency ensures faster neural response times.

NODE

A specific processing unit or server cluster within the Nexus infrastructure where commands are executed.

PROTOCOL

The fundamental set of cryptographic rules that define how data and commands are securely transmitted across the network.

UPLINK

A primary communication bridge between the Nexus core and external API systems (e.g., Discord Gateway).

FIREWALL

An active defensive barrier that monitors incoming data vectors to protect sectors from unauthorized intrusions.

NEURAL LEVEL

A classification metric representing an operative's clearance level, earned through active contribution and data exchange.

BREACH

A controlled security bypass often initiated during high-stakes simulation exercises or cyber-heist operations.

GATEWAY

The primary entry point for all neural transmissions to the Discord API. The Gateway manages real-time event streams.

WEBSOCKET

A persistent, bidirectional data stream protocol used for real-time communication between the Nexus and Discord.

PACKET

A discrete unit of data transmitted over the network. Each command, event, and message is wrapped inside a packet.

HANDSHAKE

The initial cryptographic greeting between two nodes, establishing a secure communication channel.

SSL / TLS

Secure Sockets Layer — encryption protocol ensuring data safety during transit between the Nexus and external endpoints.

CLUSTER

A collection of shards operating on the same hardware node. Clusters enable distributed processing at scale.

API KEY

A cryptographic token used for authorized access to external services and the Nexus core itself.

WEBHOOK

An automated data push mechanism that forwards events to a specific endpoint/channel in real time.

Financials

Economy Protocols

Understanding the credit exchange and asset management matrix.

💳 Credit Reserves

Credits are the primary unit of exchange. Earned via labor packets (/work) or daily allocations (/daily with streak bonuses!), they are stored in encrypted vaults linked to your User ID.

🥷 The Robbery Matrix

Operatives can attempt to breach the credit vaults of others. This carries high risk—failed breaches result in significant credit loss. Shield Modules offer 24hr protection.

🎒 Inventory Storage

The shop has 8 items: Lucky Charms, XP Boosters, Shield Modules, Crate Keys, and more. Items can grant temporary buffs to breach success rates or credit yield.

📈 Streak System

Daily claims build a streak counter. Each consecutive day adds +100 CR to your daily reward, capped at +2000 CR. Missing two days resets your streak to 1.

🎰 Casino Matrix

Slots and Blackjack offer high-risk credit multiplication. Slots support Lucky Charm bonuses. The jackpot (3x 🎰) pays 50× your wager.

🧬 Level Scaling

Work pay scales with your Neural Level. Higher levels earn better gigs with a bonus of Level × 25 CR per job. Invest in XP Boosters for faster progression.

Security Architecture

Multi-layered defense system protecting all Nexus operations.

🚫

Blacklist Gate

Severed operatives are permanently blocked from executing any command across all sectors.

🏛️

Guild-Only Lock

All slash commands are restricted to server channels. DM execution is rejected at the protocol level.

🔐

Owner Gate

Root-level commands are hardcoded to a specific user ID. Unauthorized access returns ACCESS_DENIED.

⏳

Cooldown Engine

Per-command, per-user rate limiting with category-based durations. Owner bypasses all cooldowns.

🔇

Anti-Spam Shield

Message-level flood detection auto-deletes rapid transmissions. Memory auto-prunes every 60 seconds.

🛡️

Module Toggles

Server admins can disable entire command categories (Economy, Casino, Fun, Leveling) per-server.

🧹

Word Filter

Configurable per-server bad word list. Flagged messages are auto-deleted with a warning embed.

🔄

Error Isolation

Double try-catch pattern prevents any single command crash from killing the entire bot process.
